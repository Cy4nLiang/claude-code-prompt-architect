# 多镜头时间轴与跨镜一致性

> 数据截至 2026-06，各家迭代快，**以官方最新文档为准**。
> 核心结论：**时长 = 信息预算**。动作数、镜头数、台词字数都由总时长反推：一个简单动作约 1.5–2.5s，一个镜头 1.5–7s，一句短台词 2.5–3.5s。先用预算定下"单镜头还是多镜头、几镜"，再动笔写词——预算装不下的内容，写得再好也会被模型挤掉或赶戏。

## 一、总时长 → 叙事结构映射

| 总时长 | 镜头预算 | 叙事弧 | 分段建议（当信息预算比例理解，勿当字面秒数） |
|---|---|---|---|
| **4s** | 1–2 镜 | 冲击→推进→收束 | 开场即钩子 → 单一主动作推进 → 高潮定格 / 切黑收尾 |
| **8s** | 2–3 镜 | 钩子→建立→升级→高潮 | 前 1/4 抓注意力 → 交代环境或人物 → 动作/情绪升级 → 末段高潮或反转 |
| **10s** | 3–4 镜 | 抓→落地→升→峰→收 | 钩子 → 建立（慢节奏让观众入戏）→ 首个动作（中速）→ 升级（快）→ 高潮+情绪落点 |
| **15s** | 4–6 镜 | 完整弧：钩→立→人→升→峰→收 | 钩子 → 环境建立 → 人物/主体入场 → 冲突升级 → 高潮 → 收束镜头（情绪着陆的最后一帧） |

- 节奏通则：开头慢（观众入戏）→ 中段加速 → 高潮最快或骤停 → 结尾回落。
- 音频弧线跟随叙事弧：环境音起 → 主旋律进入 → 层次渐强 → 高潮卡点 → 收尾淡出。8s 以上值得显式写。
- ⚠ 各档逐段划分主要来自单一教程信源（higgsfield Seedance 系），叙事弧本身是通用剪辑常识；用它分配内容多少，不要把分界秒数原样写进 prompt（除非厂商支持 timecode，见 §4）。

## 二、单镜头时长启发式（按内容类型）

| 内容类型 | 建议时长 | 默认景别 |
|---|---|---|
| 快动作 / 突发事件（急刹、奔跑、爆发） | 1.5–2.5s | medium shot（中景） |
| 复合动作序列 | 2.5–4s | medium shot（中景） |
| 对话 / 表情反应 | 2.5–7s 按台词长短：短句 2.5–3.5s / 中等 3.5–5s / 长段情感 5–7s | close-up（特写）/ over-shoulder（过肩） |
| 环境 / 场景建立 | 3–5s | wide shot（全景） |
| 情感高潮 / 关键台词 | 4–6s | close-up（特写） |
| 特写细节 / 道具揭示 | 2–3s | close-up / extreme close-up（极特写） |

- 台词在镜头描述里**逐字写全**，不要写"她抱怨了一句"——模型只会生成你写出来的话。
- 台词长度预算：一个 8s 片段一两句短句为宜（英文约 12–15 词）⚠（Veo 信源经验值，其他家方向一致：短剧类 prompt 普遍要求"每句台词控制在短句"）。

## 三、长内容拆分规则

**何时拆**：内容超出厂商单段生成上限，或单镜信息密度过高（>7s 还塞着多个视觉焦点）才拆。<3s 不拆；3–6s 只在有自然停顿点时拆；拆出的每段都要有独立的视觉焦点或情绪递进。

| 维度 | 规则 |
|---|---|
| 动作切点 | 动作完成后的瞬间、转身前、视线转移时 |
| 台词切点 | 句号 / 省略号后、情绪转折处；**绝不在句中切断**，整句必须完整落在某一段里 |
| 情绪切点 | 呼吸停顿处、表情变化前 |
| >2 段结构 | 首段建立状态（约 30%）→ 中间段发展变化（约 40%）→ 末段收束情感（约 30%） |
| 信息分配 | 首段含起始状态，后续段含变化过程；所有段描述合并后等于原始全部关键信息，不丢失、不新增 |
| 音频对齐 | 台词结束点与片段结束点对齐；环境音可跨段延续 |

**文戏延长、武戏拼接**：单一场景内的长对话、情绪递进、单一路径移动 → 用"视频延长"做连续长镜头（沉浸、一镜到底感）；剧情转折、追逐打斗、蒙太奇 → 独立生成片段再剪辑拼接（保节奏与冲击力）。实战常混用：先延长生成对话主体，再拼空镜与转场片段。

## 四、多镜头骨架写法

**厂商无关默认**：用 `镜头1 / Shot 1` 标号、按事件发生顺序排列，每镜写满 4 个维度：

1. **运镜 / 切换方式**：slow push-in（缓慢推近）、cut to（切至）、static shot（固定机位）
2. **主体动作与表情**：核心角色的关键动作与神态变化
3. **空间 / 位置变化**：主体所处场景、与环境和其他角色的空间关系
4. **音频**：该镜的音效 / 台词 / 配乐变化

衔接规则：wide→medium 定空间，medium→close-up 聚情绪，close-up→close-up 走视线匹配；同一景别别连用超过 3 次。

**timecode 条件分支**（精确秒数只对明确支持的厂商写）：

| 厂商 | 写法 |
|---|---|
| Seedance 2.0 | `[00:00-00:05] Shot 1: ...`——社区共识写法，样例库中最高频模式 |
| HappyHorse | `SHOT (0:00-0:05): ...` ⚠ |
| Kling | Multi-Shot 模式（约 6 镜 / 15s）⚠ |
| Veo | 顺序措辞更稳：`first... then... finally...`；timecode 支持 ⚠（信源相互矛盾） |
| Wan 及其他 | 用默认"镜头N"写法，不写秒数 |

⚠ 多数模型对精确秒数支持不稳定，强写时长可能导致生成异常——默认**不写秒数**，让模型按剧情自然分配节奏；秒数只在上表确认支持的厂商里用。

## 五、转场写法

| 转场 | 适用 | 英文措辞示例 |
|---|---|---|
| 硬切 hard cut | 情感高潮、节奏冲击 | `hard cut to ...` |
| 淡入淡出 fade | 场景变化 | `fade out, fade in to the next scene` |
| 叠化 dissolve | 时间跳跃 | `dissolve to ...` |
| 跟拍 / 摇移过渡 | 视角转换、同场景换主体 | `camera pans from A to B in one continuous move` |
| Whip pan 转场 | 同能量场景间无缝换景 | `whip pan from subject A to subject B, motion blur obscures the cut` |
| Match cut（形状/运动匹配） | 两镜元素同形或同轴旋转 | `the spinning ring of flames seamlessly transforms into a rotating ice blade, camera follows the rotation axis` |
| 粒子 / 元素连续性 | 高动态场景跳转 | `the shattered ice crystals turn into drifting red leaves; the camera passes through the fragments and cuts to the next scene` |

## 六、跨镜头一致性三档解法

| 档位 | 做法 | 适用 | 措辞示例 |
|---|---|---|---|
| **咒语级**（单次生成内） | 全局段加一句一致性约束 | 所有厂商的多镜头单 prompt，成本最低 | `characters maintain consistent faces, clothing, and hairstyles throughout, no deformation or drift`（人物一致性稳定，脸不变、服装不变） |
| **文本逐字复述级**（跨多次生成） | 角色描述固定成模板句（姓名+年龄+发型+眼睛+服装+体态），每条 prompt **逐字原样复用**；再加 signature elements（标志性配饰 / 习惯动作）强化辨识 | 无参考图机制的纯文生视频（如 Veo 文生模式） | `Sarah, a 32-year-old woman with shoulder-length black hair in a bob, wire-rimmed glasses, charcoal gray blazer...`（每镜重复）+ `he always wears his vintage leather jacket and silver watch in every scene` |
| **素材契约级**（跨多次生成，最强） | 参考图锚定：角色 turnaround sheet（正/侧/背全身三视图）+ 锁定指令；素材按职责配置：角色锚定 1–2 图 + 场景定调 1 图 + 运镜参考 1 视频 + 音频 1 段 | 支持参考素材的厂商（Seedance @图片、Kling、Higgsfield @material 等） | `[CHARACTER LOCK] Use ONLY the character turnaround sheet (front/back/side full-body views) from the reference image. Preserve the exact character identity; infer motion and expression while maintaining complete visual consistency.` |

- 三档可叠加：素材契约打底，prompt 里仍写咒语级约束兜底。
- 参考素材别堆满上限：过多素材会让模型分不清特征优先级，易风格冲突、主体识别模糊。
- 工程化升级 ⚠（单一信源的结构化实践）：把一致性做成显式"契约"数据——角色（外观锚点、服装状态序列、持有道具）、场景（地点、时段、天气、在场角色、道具状态）、全局风格锚点；生成每镜 prompt 时从契约取值，而不是每次现写。
- 剧集级 / 数字人跨场景：单条 prompt 不够，走流水线——character sheet 多角度卡 → 多图参考扩展场景 → 视频模型驱动。

### 一致性自检（出多镜头 prompt 前过一遍）

1. **角色锚点**：每个出场角色在每一镜的描述里都带同一套外观锚点（服装颜色 / 发型 / 标志性配饰）？没有用裸代词"他 / 她"指代？角色缺席几镜后再出场时重新写全锚点？
2. **道具与台词**：同一道具在所有镜头中名称、颜色、表面文字逐字相同？每句台词完整落在某一镜内、没被句中切断？
3. **时空连续**：相邻镜头的时段 / 天气 / 光源一致？景别衔接符合 wide→medium→close 的递进逻辑、没有同景别连用超过 3 次？
