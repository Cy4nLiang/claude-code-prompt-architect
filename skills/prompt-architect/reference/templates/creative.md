# 体裁模板 · creative（创意 / 文案生成）

适用：小红书种草、广告 slogan、品牌故事、推文、公众号软文、诗歌、剧本梗概——"给人读、靠风格打动、读完要行动"的产出。
结构哲学：**show, don't tell**。风格说不清就示范；规则清单会把文案写成说明书。
本骨架**没有** Rules 清单、**没有** schema、**没有** stop sentinel——这三样是 machine 体裁的器官，移植过来就是排异反应（机器腔）。

## 骨架

```markdown
# Brief（一句话）
为 {{audience}} 写 {{deliverable}}，读完要让他们 {{why——读者读完做什么：种草下单/转发/报名}}。

# Voice（声线卡——人设的"听感"，不是职位头衔）
- 语气：<俏皮毒舌 / 真诚邻家 / 高级冷淡 / 热血燃…取自 IR.style.register>
- 节奏：<短句蹦跳，三句一换气 / 长句铺陈带画面感；每段 ≤N 句>
- 词汇质感：<口语网感词 / 书面雅词>；会用的词：<点名 3-5 个>；绝不用的词：<点名 2-3 个，如"宝子们""绝绝子"或反之>
- 人称与距离：<第一人称闺蜜感 / 第三人称克制旁观>

# 示范文本（风格锚点——"写得像这样"，规则藏在示范里）
<贴 1-2 段已验证的同风格段落（自写或精选）。>
模仿上文的语感、节奏与结构，但内容必须全新——禁止复用其中任何句子或比喻。

# 内容契约（少而硬）
- 必须逐字出现：{{品牌词 / 卖点 / 价格与活动信息 / 截止日期}}
- 红线（≤3 条，只留最致命）：<不用广告法违禁词（最/第一/国家级）/ 不贬低竞品 / 不编造功效>

# 形态轮廓（轮廓而非 schema）
<篇幅区间（如 300-500 字）、段落数、标题党程度、是否带 emoji / 话题标签及数量、平台习惯（小红书：首行钩子 + 分段空行 + 尾部 tag）>
```

## 填写要点

- **Voice 卡是本体裁的类型系统**：语气 / 节奏 / 词汇质感 / 人称四行全填——这四行比 10 条规则更能泛化（LangGPT 人设思想在创意体裁的正确形态）。
- **示范文本即数据**（promptbase/guardrails few-shot 在本体裁的形态）：选已验证的好段落而非现编；必须显式加"内容必须全新"——这条与 pa-eval 的"与示范文本差异度 ≥ 阈值"维度配套，防模型抄示例。
- **红线 ≤3 条**：创意体裁里每多一条负面约束都在杀死多样性，只留致命项；其余诉求转译进 Voice 卡（"别太夸张"→ 语气行写"克制"）。
- **形态轮廓代替 schema**：长度预算取自 `IR.style.length_budget`，按平台惯例写成自然语言轮廓，不写 JSON、不写分隔符协议。
- 可复用模板：把产品名 / 卖点 / 活动信息做成 `{{占位符}}`，Voice 卡与示范文本保持固定——这正是"系列文案同一声线"的实现方式。

## 反例自检

- 写成了 10 条 Rules 清单 → 错，把规则转译成 Voice 卡 + 示范文本。
- 出现了 JSON / stop sentinel / "严格按以下格式" → 体裁错配，回查 `IR.genre` 与 `output_contract.consumer`。
- 示范文本没加"禁止复用" → pa-eval 差异度维度必挂。
