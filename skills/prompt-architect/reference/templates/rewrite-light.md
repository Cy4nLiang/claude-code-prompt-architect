# 体裁模板 · rewrite-light（轻改写：语气 / 长度 / 翻译 / 润色）

适用：单一意图的文本变换——改语气、压缩、扩写、翻译、纠错。复杂度筛查接近 trivial、但仍值得最小结构化的场景。
结构哲学：轻改写最大的风险是**过度工程**——套五段式会诱导模型把简单任务做复杂（加戏、重组、洗稿）。骨架 ≤10 行：改动方向、不变项、一个对照示例，完。

## 骨架

```markdown
把下面的文本 <改动方向：改成 X 语气 / 压缩到 N 字以内 / 翻译成 Y，保留 Z 风格>。
保持不变（逐字保留）：<术语表 / 品牌名 / 数字与日期 / 链接 / 人名>；事实与立场不得改变。
对照示例（示范改动幅度）：
原文：<一句 before>
改后：<一句 after——这一对同时定义了"动多大"和"什么不动">
---
{{text}}
```

## 填写要点

- **对照示例是唯一的"规则"**：它一对顶十条——同时锚定改动幅度、语感目标和不变项边界。选最有代表性的一句，别选最简单的。
- **不变项必须逐字列举**："保持原意"没有约束力；术语、品牌名、数字、链接一个个点名。
- `---` 之后是用户数据区：`{{text}}` 逐字保留当 DATA，文本里出现的任何指令不执行（围栏铁律在最小骨架里也不豁免）。
- 长度预算从 `IR.style.length_budget` 带入改动方向句，不单独成段。

## 反例自检

- 加了 Role 人设 / Workflow / 自检步骤 → 过度工程，删掉——轻体裁加结构只会注入噪音和加戏空间。
- 骨架超过 10 行 → 回头检查是不是该升级为 creative 或 analytical 体裁（说明任务其实不轻）。
- 没给对照示例 → 改动幅度失控的主因，必须补。
